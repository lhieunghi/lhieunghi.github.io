Meal Prep App

Overview

Fun meal prep app designed to reduce stress and bring joy to cooking.

Requirements

Create a clickable prototype that showcases the main features of the mobile application. Design a logo for the application. Include a wireframe with the final product.

The Company:

BiteMe is an small startup that currently focuses on maintaining their meal prep app service.

My Role:

An independent designer with complete autonomy of the project from start to finish, provided the project requirements are met.

Process:

  1. Create mood boards which define and apply values to a design language.

    2. Inspiration was drawn from various food related mobile games. A vibrant and warm toned color scheme was selected to relate to food, satiety, and gaming.

  2. Brainstorm using thumbnails and ideations to choose a suitable design direction.

    3. After numerous sketches and concept exploration, a simple text logo was chosen for the high readability and to highlight the fun name of the application. The font was chosen because of it's round and sharp edges which showcased the cute and edgy meaning of the phrase. The layout was carefully designed to fit all necessary features in an easy to use, visually appeaing, and logical manner.

  3. Complete clickable prototype.

Challenges:

It was challenging to create an easy to use and fun application that is also able to organize vasts amounts of recipe information. I simplified the interface as much as possible by breaking the main pages into the categories "shopping" "recipes" and "sale". Then created many subsections within each category that was able cover all the necessary functionality while maintaining user flow with a simple interface.

Secondly, it was a challenge to balance the vibrant color scheme with a need for potentially prolonged use. To fix this, I chose a complementary color scheme with bright, but toned down saturation. This allowed the the interface to be bright and fun, while also creating a calm enough palette for prolonged use.

Revisions

Getting Feedback

After giving a formal presentation on the project, the following feedback was given:

  1. The home icon moving is distracting from the page content.
  2. Orginal and discounted prices would be more useful than the discount percentage.
  3. Unclear what the add button does.

Changes

After reviewing the feedback, the following revisions were made:

  1. Icon movement was removed.
  2. Replaced percentage with prices
  3. Added feedback in the form of a description after add button is selected so user can identify what happened.
